This interface lets you bulk-import assets into this database. You will need a spreadsheet meeting the following criteria:

  • The first row must consist of a name for each column, selected from the list below. The name must exactly match the name in the list.
  • The spreadsheet needs to be in CSV format (comma-delimited), so you have a .csv file.
  • No blank rows or columns in the body of the data.

Valid column names

Mandatory columns must appear somewhere in the spreadsheet. Other columns are optional and may be excluded.

Column nameDescription
ASSET TAG (mandatory) The asset tag of the item (in form ITXXXXX).
SERIAL (mandatory) The asset's serial number.
DATE PURCHASED (mandatory) The purchase date for this asset. Must be in the form dd/mm/yyyy.
WARRANTY END (optional) The end date of the warranty for this asset. Must be in the form dd/mm/yyyy.
FINANCE ASSET TAG (optional) The asset tag of the item as issued by Finance Branch.
VENDOR (optional) The vendor name of the product (eg. Dell).
MODEL TYPE (optional, must match existing value) The hardware model type (eg. Server - Rackmount).
HARDWARE MODEL (optional) The hardware model name (eg. Latitude E6320).
STATUS (optional) The current deployment status of this asset. If not provided, this will be set to 'In storage'.
COST CENTRE (optional) The cost centre to which is asset belongs.
LOCATION (optional, must match existing name) The name of the location of this asset.
PURCHASED VALUE (optional) The amount paid for the asset, inclusive of any permanent modules or upgrades.
ASSIGNED USER (optional) The username of the user this asset is assigned to.
NOTES (optional) Any additional notes regarding this asset.
